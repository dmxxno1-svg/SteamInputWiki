# 06b01：判断 Steam Input 是否适合你的项目

本章并非贬低混合输入或原生 Steam Input 的价值；理想情况下两者兼顾，但若只能择一，请根据项目需求做出判断。

以下几点可帮助决定：

- 小团队可能难以投入大量资源实现完整 SIAPI，此时可优化对混合输入的支持并提供易用的配置界面。
- 若游戏设计本身非常简单（比如平台跳跃类），过度实现 SIAPI 可能收益有限。
- 若不打算以 Steam 为主要发行平台，SIAPI 的收益会受限（SIAPI 在 Steamworks SDK 中可用）。

若决定不以 SIAPI 为主，可以考虑其他输入选项（DirectInput、XInput、引擎自带输入系统、SDL2 等），并确保对混合输入与通用可访问性做支持。

## 优化混合输入的建议

* 玩家不应因混合输入而出现明显问题（帧率、卡顿或输入受限等）。
* 将瞄准辅助设为可选并与输入设备无关。
* 提供锁定提示图标的选项，帮助混合输入用户获得稳健体验。
