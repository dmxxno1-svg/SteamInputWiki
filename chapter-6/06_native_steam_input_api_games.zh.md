# 第06章：原生 Steam Input API 游戏

正如 SIAPI（Steam Input API）简介中提到的，有些开发者在其游戏中实现了 SIAPI，这就提供了“原生”的 Steam Input 支持，也即“原生”的 Steam 控制器支持。

在游戏中实现 SIAPI 有许多好处，例如：
- 绑定动作而不是物理按键
- 根据游戏上下文自动切换动作集（Action Set）
- 游戏内按键提示会始终与玩家使用的控制器及其配置自定义保持一致
- 以及其它许多便利功能

但这并非完美无缺。作为一种相对较新的输入 API，部分开发者实现良好，而另一些则存在问题。实现质量是一个光谱，玩家在购买前若能了解某款游戏的 SIAPI 实现质量会很有帮助。

本章我们将制定一个评分体系来评估 SIAPI 的实现质量，并据此对游戏进行评分。目标有两点：

* 为玩家提供参考，帮助做出明智的购买决策；
* 为考虑实现 SIAPI 的开发者提供示例和最佳实践，帮助他们避免常见错误并设定实现目标。

此外我们还会为开发者提供实现 SIAPI 的资源，包括：

* 如何判断 SIAPI 是否适合你的游戏 —— 虽然大多数游戏都能受益，但对部分项目来说实现成本可能高于收益
* Valve 的官方文档链接
* 从玩家角度总结的“最佳实践”指南：哪些做法能让玩家获得更好的体验
