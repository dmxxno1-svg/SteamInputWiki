# 06a01：SIAPI 实现评分体系

本章致力于制定一个体系，用来评估开发者在游戏中实现 SIAPI 的质量。评估目的不是贬低开发者，而是给玩家提供信息并帮助开发者改进实现。

每款游戏从相同基准分开始，然后根据多个类别进行加减分，最终得出评分并映射到评级。

## 评估类别示例

* 游戏在使用 SIAPI 时是否可玩？若不可玩则自动给最低评级。
* Action Sets（动作集合）：命名是否清晰、覆盖所有上下文、自动切换是否可靠等。
* 可绑定动作（Bindable Actions）：动作命名与数量是否合理、是否能与其它输入系统保持技术等价性。
* 可绑定输入风格（Bindable Input Styles）：是否提供 `absolute_mouse` 风格、是否有不当将动作硬编码为风格等。
* 其他问题（帧率问题、硬编码输入、自动 Action Layers 的滥用等）会扣分。
* 若 SIAPI 带来独特功能（如特殊触觉、原生陀螺支持等），可加分。

## 最终评级示例

- Copper（铜）：≤ 70，体验欠佳或不可用
- Silver（银）：71–80，基本可玩但有问题
- Gold（金）：81–90，运行良好
- Platinum（白金）：91–100，几乎无缺陷
- Diamond（钻石）：>100，超出常规实现并实现独特特性
