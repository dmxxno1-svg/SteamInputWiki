# 05d：基于配置的触觉（Configuration Based Haptics）

## 什么是触觉反馈？

触觉反馈是一种仿真物理反馈，有时称为 HD Rumble 或 3D Touch。在 Steam Controller 上，触觉由两个内置在双触摸板下方的执行器产生，可模拟非常细腻的振动或脉冲感。尽管 Switch Pro 与 DualSense 也有触觉马达，但对 Steam Input 的完整支持有限，因此本章主要以 Steam Controller 为例讲解。

触觉旨在提供比传统 Rumble 更细腻的沉浸感，同时也用于为触摸板或其他仿真输入提供关键性的触感反馈。

随着 Steam Input 的功能增强，用户现在可以更灵活地在配置中控制触觉效果。本章覆盖如何在配置中创建用户定义的触觉效果与技巧。

## 给开发者的说明

如果你是开发者且对原生触觉有兴趣，建议参考 Valve 官方文档以获取更底层的控制接口。本章提供玩家视角下的实用建议，便于设计更好的 SIAPI 支持。

## 创建用户定义触觉

本节讨论将绑定与 Activators 用于触觉创作的策略（这些用法侧重于触觉而非常规输入映射）：

- 空绑定（Empty Bindings）：用于在不向游戏发送输入的情况下触发触觉。
- 常规按下（Regular Press）：可在按下时触发触觉，使其像常规控制器的反馈一样工作。
- 软按（Soft Press）：在跨过模拟阈值时触发触觉，用于在触摸板或触发器上创建“虚拟环”或“虚拟扳机分级”感受。
- 释放按下（Release Press）：在释放时触发触觉，可用于模仿真实设备的释放感。
- 和弦按下（Chorded Press）：在按住特定输入时替换或增加触觉效果。

### Activator 与触觉特性

不同类型的 Activator 会呈现出不同的触觉特性；例如 Soft Press 的触觉较强但只适用于被视为模拟的输入类型。Turbo（按住重复）可用于以固定频率持续脉冲触觉，通过组合空绑定和延迟可以实现更复杂的触觉节奏。
