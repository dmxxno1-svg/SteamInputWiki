# 第 05f 章：作为开发者兼顾输入辅助功能

本指南并非责备未考虑或无法兼顾各种辅助选项的开发者；我们理解预算、人力或设计愿景可能限制取舍。目标是鼓励在可行范围内，让更多玩家享受游戏。

请注意：不应仅依赖 Steam Input 提供辅助功能。输入相关的可访问性选项（重映射、死区、按键替代等）最好内建在游戏中，而非完全交由第三方重映射工具来实现。

## 触觉作为替代反馈

对于视觉或听觉受限的玩家，触觉（传统 Rumble 或更高级的 Haptic）可作为替代或补充反馈。例如在 FPS 中通过触觉指示被击方向，或在开放世界中通过脉冲提示目标方向。建议将此类选项作为可配置项，并允许关闭或调整强度。

## 对感官敏感玩家的处理

触觉可能对部分玩家造成不适，开发者应提供完整关闭或降低强度的选项。若使用 Steam Input 的触觉功能无法完全关闭，需在原生实现层面提供替代方案。

## 对运动功能受限玩家的考虑

应避免强制玩家在短时间内完成多项按键操作，提供将按住改为切换、可调容错时间等选项。对于需要高频操控的 QTE（快速时间事件），应提供跳过或放宽失败条件的选项。

## 瞄准辅助与可访问性

建议将瞄准辅助做为可选项并允许跨输入设备使用（而非仅对手柄启用）。为单/合作模式提供选项可以显著提升可访问性。

## 运动控制（Gyro）的可及性考量

陀螺可提升某些玩家的体验，但也可能对他人造成不便。应将陀螺支持做为可选，且绝不可作为唯一途径来完成关键玩法。

## 支持 Copilot / 多控制器与自适应设备

混合输入（Mixed Input）亦可指多个控制器并用。建议确保游戏能容纳多个输入源同时作为 Player 1 输入，以更好地支持自适应设备与 Copilot 等功能。

## 优化自适应技术的实现建议

将可访问性选项置于设置菜单中，而非只在检测到自适应设备时自动启用；给玩家选择与控制权很重要。

## 推荐资源

建议查阅 https://accessible.games 以获取更多辅助功能设计思路。

# 第05f章：作为开发者在输入方面照顾无障碍

本指南并非为指责未考虑无障碍的开发者而写；我们理解有预算与实现上的限制。目标是鼓励开发者在可能时为更多玩家提供可访问的体验。

重要提醒：不要仅依赖 Steam Input 来实现无障碍功能。输入重映射等核心无障碍选项应尽量内置于游戏中，以便在玩家没有特定第三方软件时也能使用。

## 使用触觉作为替代反馈

对于有视觉或听觉障碍的玩家，触觉可以作为替代或补充反馈（例如朝被射击方向触发触觉，或用于指引目标方向）。建议将此类功能作为可选项置于辅助功能菜单，并允许关闭或调节强度。

## 为有感官敏感问题的玩家处理触觉

考虑到少数玩家可能对触觉感到不适，建议为原生触觉功能提供关闭项或更低强度选项。Steam Input 在某些情况下无法完全关闭触觉，开发者若需保证可完全关闭，应在原生层面覆盖触觉输出。

## 针对动作/灵活性受限的玩家

提供将长按改为切换（或反之）、降低对同时复杂按键组合的依赖、允许跳过或简化快时事件（QTE），以及允许调整按键重复率与死区，这些都能帮助有动作受限的玩家更顺利地游玩。

## 瞄准辅助作为无障碍选项

在单人或合作游戏中，应考虑将瞄准辅助作为独立选项而非强制绑定到某种输入设备。建议为该功能提供开/关与强度选项，并考虑为所有输入源提供统一设置。

## 运动控制（陀螺）与无障碍

陀螺有时是有益的无障碍功能，但也可能对部分玩家造成负担。务必将陀螺支持做成可选，并提供替代方案以实现同等的游戏互动。

## Copilot 与多控制器兼容性

混合输入也可能意味着允许多个控制器同时作为玩家一来使用。确保输入系统支持多控制器与任意输入源的组合，或提供开关与选项以便适配自定义辅助设备。

## 推荐做法

将无障碍设置放在选项菜单中，避免仅在检测到某种辅助设备时自动启用功能；并尽量提供可配置项让玩家自定义体验。

## 资源

建议参考 https://accessible.games 获取更多无障碍设计思路与资源。
