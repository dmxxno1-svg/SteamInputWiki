# 第 03a 章：绑定页面结构解析

（Mennenths 的笔记：不包含在最终维基中：将会截取屏幕截图作为视觉辅助。）

当你选择一个想要分配绑定的按钮时，会进入绑定页面。

页面顶部，如同其它页面，是配置的名称。正下方显示的是该配置的作者。

屏幕上方中央会显示你当前所使用控制器的图片。

屏幕下方中央是包含所有绑定的框。

首先被高亮的是一个文本输入字段，用于为你的绑定命名。

在文本输入字段右侧是一排与 Steam、Steam Input、控制器或你的电脑系统交互的绑定按钮。

在文本输入和系统绑定下方是用于实际游戏操作的绑定区域：鼠标按键、键盘按键和手柄按键。下一节会详细介绍这些内容。

页面底部（如同其它页面）是用于与页面元素交互的快捷绑定和 UI 按钮。

- Listen for Key
  - 将 Steam Input 切换到“监听模式”。你无需在界面上逐项选择绑定，而是直接按下鼠标、键盘或手柄上的按键进行绑定。
- Remove
  - 移除当前绑定。
- Toggle Multi-Button On
  - 允许绑定多个按键组合，使你能通过一次按下激活一组组合。例如在桌面配置中可以用来方便实现 Ctrl+C / Ctrl+V 的复制粘贴。
- Show Activators
  - 进入 Activators 页面以对按钮行为做更细致的控制。Activators 在第 04 章中会介绍。
- Legacy Keys
  - 仅在已实现 SIAPI 的游戏中可用。
  - 返回默认的鼠标、键盘和手柄绑定界面。
- In Game Actions
  - 仅在已实现 SIAPI 的游戏中（查看 Legacy Keys 界面时可见）。
  - 返回 SIAPI 游戏内动作列表。

# 第03a章：绑定页面结构（Anatomy of the Bindings Page）

（注：Mennenths 的备注不应包含在最终 Wiki 中：将为视觉说明截取截图。）

当你选择一个要分配绑定的按钮时，会进入绑定页面。

页面最顶端（与其它页面一致）是配置的名称；紧接其下显示配置的作者。

页面上方中间会显示你当前使用的控制器图片。

页面中部偏下是包含所有绑定的区域框。

首先高亮的是一个文本输入框，可以在此为你的绑定命名。

在文本输入框右侧是一排与 Steam、Steam Input、控制器或电脑系统交互的绑定图标。

在文本与系统绑定之下是用于实际游戏操作的绑定区域：鼠标按键、键盘按键和手柄按键。本章后续会进一步介绍这些内容。

页面底部（与其它页面一致）是与页面元素交互的按键和 UI 按钮。

* Listen for Key（监听按键）
  * 这会把 Steam Input 置于“监听模式”。你无需在界面上逐项选择绑定，而是直接按下鼠标、键盘或手柄上的键来绑定。
* Remove（移除）
  * 移除当前绑定
* Toggle Multi-Button On（开启多键绑定）
  * 允许你绑定多个按键，使你可以通过一次按下激活一个组合。例如在桌面配置中可把 ctrl+c 与 ctrl+v 组合起来实现复制粘贴的快捷操作。
* Show Activators（显示激活器）
  * 如果你想更深入地控制按钮行为，此项会带你到 Activators 页面。Activators 会在第 04 章讲解。
* Legacy Keys（旧版按键）
  * 仅在已实现 SIAPI 的游戏中可用
  * 返回到默认的鼠标、键盘和手柄绑定界面
* In Game Actions（游戏内动作）
  * 仅在已实现 SIAPI 的游戏中且查看 Legacy Keys 屏幕时可用
  * 返回到 SIAPI 的游戏内动作列表
