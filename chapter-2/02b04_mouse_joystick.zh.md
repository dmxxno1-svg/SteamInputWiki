# 02b04：鼠标-摇杆（Mouse Joystick / 类鼠标摇杆）

推荐观看：相关演示视频（待补充）

此输入风格允许你像使用鼠标输入风格那样与设备交互，但会将鼠标移动进行运算转换为等效的摇杆角度与偏转值。目标是玩家做出类似鼠标的输入，但游戏看到的是摇杆（通常为右摇杆）并据此响应。该风格旨在作为不允许混合输入（见第 02c 章）且需要游戏手柄输入的兼容方案。

注意：因为游戏端接收到的是摇杆输入，所以此模式下的灵敏度只能在游戏内调整。

该风格可应用于：

* 触控板
* 陀螺（以下简称“Gyro”）

当应用于触控板时，你的交互方式类似于将触控板置为鼠标风格（参见第 02b03 节）。

当应用于陀螺时，你的交互方式类似于将陀螺置为鼠标风格（参见第 02b03 节）。

## 设置

### 点击操作（Click Action）

仅适用于触控板。允许你将触控板按下绑定为某个操作。例如在某些游戏里通常用右摇杆按下执行某些功能，当你用触控板模拟瞄准时，可以把触控板点击绑定为右摇杆按下以保留该功能。

### 陀螺启用按钮（Gyro Enable Button）

仅适用于陀螺。选项繁多：你可以选择哪个按钮启用陀螺，或将陀螺设为常开（Always On）。

### 陀螺按钮行为（Gyro Button Behavior - Turns Gyro）

仅适用于陀螺。选项：On（按住启用）、Off（按住禁用）、Toggle（按一次切换）。

### 灵敏度（Sensitivity）

对触控板与陀螺“可用”，但在 UI 中被灰显并提示“请在游戏内调整灵敏度”。正如 Valve 所述，最佳效果通常在游戏内把摇杆灵敏度调高。

### 旋转（Rotation）

对触控板与陀螺可见，但它似乎只影响触控板而不影响陀螺（可能是对陀螺显示但无效的 BUG）。旋转设置允许在软件中调整触控板的 xy 网格角度，以便将触控板的 x 轴与拇指的自然水平滑动对齐（参见第 02b03 节关于 Rotation 的说明）。

### 垂直灵敏度缩放（Sensitivity Vertical Scale）

用于在 x 与 y 轴之间调整灵敏度比例。与鼠标风格不同，Mouse-Joystick 将输出转换为摇杆后，确定确切比率会更困难；建议对触控板稍微压缩垂直灵敏度以帮助控制，但需要更多实验。

### 轨迹球模式（Trackball Mode）

仅适用于触控板。使鼠标部分在你“弹动（flick）”触控板后有虚拟惯性，继续使虚拟摇杆维持偏转一段时间并逐渐回中。

### 轨迹球摩擦（Trackball Friction）

仅适用于触控板。控制虚拟轨迹球减速的快慢，选项：Off / Low / Medium / High / None（永远不停止，直到你再次放下手指）。

### 垂直摩擦缩放（Friction Vertical Scale）

仅适用于触控板。控制虚拟轨迹球在 y 轴上的减速相对于 x 轴的比例。对摄像机控制类游戏，提高 y 轴摩擦有助于让水平弹动主要影响 x 轴。

（注：作者对 Trackball 在 Mouse-Joystick 下并不欣赏，因其在不同游戏上的表现不稳定。）

### Gyro 摄像机比例（Gyro Camera Scale）

仅适用于陀螺。若同时对触控板与陀螺使用 Mouse-Joystick，该滑块决定将陀螺输入在转换为虚拟摇杆时相对于触控板输入的重要性。

### 陀螺倾斜绑定（Gyro Lean Left/Right Bindings）

仅适用于陀螺。将陀螺视为大摇杆，倾斜到一定角度时触发绑定。

### 模式切换（Mode Shifting）

允许设置按键将当前设备切换为完全不同的输入风格（第 04 章详述）。

### 触觉强度（Haptics Intensity）

当鼠标模式下启用触觉时，输入检测到移动会产生短促振动，强弱随灵敏度而变。选项：Off / Low / Medium / High。

（注：除非启用 Trackball，否则作者并不推荐在 Mouse-Joystick 下开启触觉。）

### 垂直轴反转（Invert Vertical Axis）

适用于触控板与陀螺，选项：Off / On。注意与游戏内反转设置可能冲突。

### 最小摇杆 X/Y 输出值（Minimum Joystick X/Y Output Value）

适用于触控板与陀螺。滑块分别决定向游戏输出的最小虚拟摇杆偏转值；向右增大最小输出。

重要提示：如 Valve 所言，将该值上调直到小动作变得平滑，再在动作过大时下调。

### 提升小动作精度（Enhance Small Movement Precision）

适用于触控板与陀螺。用于在最小输出值调整后进一步微调小幅移动的响应，以增强精细动作的精确度。

（注：上述三个滑块对 Mouse-Joystick 的表现至关重要；它们帮助克服游戏对摇杆死区和非线性响应的处理，从而改善小动作与微调体验。）

### 自定义响应曲线（Custom Response Curve）

适用于触控板与陀螺。调整 Mouse-Joystick 向最大输出 ramp-up（响应上升）的速度；向右更激进，向左更平缓。

### 双击绑定（Double Tap Binding）及其参数

仅适用于触控板。双击绑定需要在设定时限内双击以触发，可选择是否启用提示音（Double Tap Beep）。

### 陀螺转向轴（Gyro Steering Axis）

仅适用于陀螺。选项：Yaw / Roll，决定哪种旋转控制水平移动（参见第 02b03 节）。

### 触发按键鼠标减速/滑块（Trigger Press Mouse Dampening）

若游戏无 ADS 或减速按钮，可用此项为“伪 ADS”设置按键或组合，并用滑块调整减速强度。

### 边缘旋转（Edge Spin）设置

仅适用于触控板。Edge Spin 在触控板中心为 absolute_mouse，在边缘为 joystick-like 输入，可用于长距离追踪目标。设置包括半径与速度。

### 最小移动阈值（Minimum Movement Threshold）

适用于触控板与陀螺。累积小幅移动直到超过阈值再一次性发送，用以对抗游戏的输入过滤。过大会导致跳动感。

## 总结与建议

Mouse-Joystick 是 Steam Input 中最难调教的风格之一，既需要大量微调，也极度依赖具体游戏如何处理摇杆输入。在某些游戏中它可能表现糟糕，而在某些 3rd 人称动作或平台游戏中（如 Dark Souls 或 Spyro）则能带来可接受甚至良好的体验。

作者提供了一个常见的调参流程：先把游戏内灵敏度最大化、把最小输出值滑到高位，再慢慢降低直到不产生抖动；提升小动作精度调整以找到合适的平衡。总体而言，如果游戏不支持混合输入，Mouse-Joystick 可作为可用的兼容方案，但仍不可期望达到纯 Mouse 风格的表现。
