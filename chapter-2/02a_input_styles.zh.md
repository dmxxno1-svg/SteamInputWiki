# 第2章：输入风格（Input Styles）

## 介绍

某些输入设备可以以多种不同方式使用。例如一个物理摇杆不仅可以作为摇杆，还可以模拟十字键（D-Pad）、模拟鼠标，甚至更复杂的用途（比如圆周滚动装置）。

Steam Input 将这些不同的使用方式称为“输入风格（Input Styles）”。输入风格是对物理设备行为的一组预定义配置方式。

## 哪些设备可以应用输入风格？

下列设备可以使用输入风格：
* 方向键（D-Pads）
* 面部按键簇（Face button cluster，例如 Xbox/Nintendo 的 ABXY，或 Sony 的 十字/圆/方/三角）
* 模拟摇杆（Analog Sticks）
* 触控板（Trackpads），包括 Steam 控制器式的主用触控板和 Sony 式的副用触控板
* 陀螺仪（Gyroscopes）

并非每种设备都支持所有输入风格。

## 我可以使用哪些输入风格？

### 传统模式（Legacy Mode）输入风格

这些是用于未实现 SIAPI（Steam Input API）的游戏时可用的风格。每种风格在本章后续对应小节中会有更详细的说明。

#### 无（None）

完全禁用该输入设备。如果你不需要某个输入设备，这是最简单的禁用方式，配置界面会显示该设备未被使用。

#### 方向键（Directional Pad）

提供类似传统十字键的界面。在触控板或摇杆等模拟设备上，还有额外选项可用于在有限空间内扩展移动选项。

注意：方向键风格不会对“同时按下相反方向（SOCD）”进行清理（SOCD cleaning）。这意味着除非物理设备自身阻止（如真实方向键），否则可能同时按下“左”和“右”（或“上”和“下”），Steam 会把两者都发送给游戏。现代原生 PC 游戏通常会自带 SOCD 清理，但在老旧或模拟器中可能会导致问题。

#### 按键面板（Button Pad）

提供类似传统面部按键簇（四向菱形按键）的界面。在触控板上可以配置按键的大小和相对间距。

#### 摇杆移动（Joystick Move）

提供传统摇杆的行为与输出。与方向键类似，摇杆风格也有额外选项以增强灵活性与动作密度。

#### 摇杆相机（Joystick Camera）

与摇杆移动类似，但针对第三人称相机控制进行了调整：初始接触点为虚拟摇杆中心，使其更偏“相对（relative）”而非“绝对（absolute）”风格，便于视角控制。

#### 鼠标（Mouse）

用于移动系统鼠标光标。此风格包含大量选项，如加速曲线与可选的轨迹球模式（trackball）。

#### 鼠标-摇杆（Mouse Joystick）

类似于“鼠标”风格，但不是直接发送鼠标移动事件，而是发送一系列摇杆“快速拨动（flicks）”。当游戏不接受混合输入（mixed input）时可作为最后手段使用，其表现高度依赖目标游戏如何处理摇杆输入。

#### 滚轮（Scroll Wheel）

模拟鼠标滚轮。该风格可由水平、垂直或圆周运动驱动，支持“前进/后退”滚动或按绑定循环切换。

#### 触摸菜单（Touch Menu）

通过 Steam 覆盖层在屏幕上提供网格菜单以选择不同动作，从而在单个输入上实现高动作密度（最多可达 16 个按钮）。

该风格不适用于摇杆，摇杆应使用径向菜单或快捷栏（Radial Menus / Hotbars）。

注意：此风格需要 Steam Big Picture 模式覆盖层才能正确显示；常规 Steam 覆盖层或没有覆盖层时不会生效。

#### 鼠标区域（Mouse Region）

将设备转换为在屏幕预定义区域内进行 1:1 定位的鼠标工具，适用于例如 RTS 小地图的视角移动。

#### 径向菜单（Radial Menu）

与触摸菜单相似，但以圆形模式显示单独按钮，适用于摇杆。触控板也可使用，取决于偏好。最多可提供 20 个按钮。

此风格不适用于方向键/面部按键簇；这类设备应使用快捷栏（Hotbar Menus）。

注意：此风格需要 Steam Big Picture 覆盖层才能正确显示。

#### 单键（Single Button）

将触控板作为单个大按键使用。

#### Flick Stick（拨动摇杆）

由 YouTuber 与开发者 Jibb Smart 创造，将摇杆或触控板变为“绝对旋转设备”。向后拉摇杆会令角色转 180 度，向左推会转 90 度，等等。该风格通常配合陀螺瞄准用于 3D 动作游戏，而非单独使用。

（注：Steam 控制器触控板可以应用该风格，DualShock 4/DualSense 也可，但握持人体工学上不太适合。）

#### 方向滑动（Directional Swipe）

类似方向键，但需要一定的模拟输入行程才会触发动作。这种“行程门槛”使其在陀螺等场景下更一致，因此常用于与陀螺配合的用例。

#### 快捷栏菜单（Hotbar Menu）

为方向键/面部按键簇提供屏幕可滚动的横向快捷栏。触控板与摇杆也可使用，最多支持 16 个按钮。

注意：此风格需要 Big Picture 覆盖层才能正常工作。

### SIAPI 模式（游戏原生）输入风格

原生实现 Steam Input API（SIAPI）的游戏可以提供其特定于游戏的输入风格。大多数传统模式（legacy mode）输入风格会被隐藏，但通过在页面底部启用“高级设置（Advanced Settings）”，适用的传统风格会重新显示，同时若游戏提供了额外设置，那些设置也会出现。

Valve 还为 SIAPI 风格补充了对部分传统风格的重新实现（例如为摇杆类风格提供“鼠标-摇杆”伴随风格），以便兼容那些未遵循最佳实践或实现有缺陷的 SIAPI 支持。

# 第 2 章：输入样式

## 简介

某些输入设备可以以多种不同方式使用。物理摇杆不仅能作为摇杆使用，还可以模拟方向键（D-Pad）、模拟鼠标，或更复杂的用途，比如圆形滚动装置。

Steam Input 将这些不同的使用方式称为“输入样式”（Input Styles）。所谓输入样式是对物理设备可以被配置为的预定义行为方式。

## 哪些设备可以应用输入样式？

以下设备可以使用输入样式：
* 方向键（D-Pads）
* 面部按键簇（面向 Xbox / Nintendo 的 ABXY，或索尼的十字/圆圈/方块/三角）
* 模拟摇杆
* 触控板，包括 Steam 控制器式的主触控板和索尼式的辅助触控板
* 陀螺仪

并非每个设备都支持所有输入样式。

## 可用的输入样式有哪些？

### 传统模式（Legacy Mode）输入样式

这些样式适用于未实现 SIAPI（Steam Input API）的游戏。各样式在本章后续小节中会有更详尽的说明。

#### None（无）

完全禁用该输入设备。如果不需要某个输入设备，这种方式最简单，能够在配置界面中明确标记该输入未被使用。

#### Directional Pad（方向键）

提供类似传统方向键的接口。对于触控板和摇杆等模拟设备，会有额外选项以便压缩更多移动选项。

注意：方向键样式不会进行 SOCD（同时按下相反方向）的清理。也就是说，除非物理设备本身阻止（真实 D-Pad 会），否则可能同时发送“左”和“右”（或“上”和“下”），这在某些老旧或模拟器游戏中可能导致问题。

#### Button Pad（按键面板）

模拟传统的四按键菱形布局（ABXY / BAYX / 十字-圆-方-三角）。在触控板上可配置按键的大小与相对位置。

#### Joystick Move（摇杆移动）

输出传统摇杆风格的输入。与方向键类似，此样式也提供额外选项以提升灵活性与按键密度。

#### Joystick Camera（摇杆相机）

与 Joystick Move 类似，但为第三人称相机控制微调：把接触点视为虚拟摇杆中心，使其更偏向相对输入而非绝对输入。

#### Mouse（鼠标）

把设备变为 1:1 的鼠标控制（软件上称作 absolute_mouse），用于移动系统光标或在 FPS 中控制相机。包含大量设置，如加速曲线与可选的轨迹球模式。

#### Mouse Joystick（鼠标→摇杆）

把鼠标式输入转换为一系列摇杆“抖动”事件，以在不能接受混合输入（Mixed Input）的游戏中提供兼容性。该样式依赖于游戏如何处理摇杆输入，通常为最后手段。

#### Scroll Wheel（滚轮）

模拟鼠标滚轮，可由水平、垂直或圆形动作驱动。支持“前进/后退”样式或在列表中循环的多项绑定。

#### Touch Menu（触控菜单）

通过 Steam Overlay 在屏幕上显示网格菜单来选择不同动作，让单一输入支持高密度按键（最多 16 个）。该样式不适用于摇杆（摇杆请使用 Radial Menus 或 Hotbars）。

注意：此样式需要 Big Picture Mode 的覆盖层才能正确显示。

#### Mouse Region（鼠标区域）

把设备映射为屏幕上一段预定义的区域，使设备在该区域内以 1:1 的方式定位鼠标，适用于如 RTS 小地图之类的场景。

#### Radial Menu（环形菜单）

类似触控菜单但呈环形，更适合摇杆使用，最多支持 20 个按钮。D-Pad/面部按键请使用 Hotbar。

注意：需要 Big Picture Mode 覆盖层来显示。

#### Single Button（单一按键）

把触控板变为单个大按键，可作为触摸或点击使用，并支持二段式行为（触摸/点击分别绑定）。

#### Flick Stick（Flick 摇杆）

由 Jibb Smart 发明，适合与陀螺 aiming 配合使用。把摇杆变为“绝对旋转装置”：向后拉可转 180°，向左/右为 90°，并在倾倒后允许按角度平滑旋转。通常与陀螺结合使用以补足垂直轴瞄准。

#### Directional Swipe（方向滑动）

类似方向键，但基于手势：通过滑动触发一次性方向动作，适合模拟触摸滑动输入。

#### Hotbar Menu（快捷栏菜单）

为 D-Pad / 面部按键等提供可滚动的横向菜单，适合快速在多项绑定间切换，最多支持 16 个按钮。

注意：Hotbar 与 Touch/ Radial Menu 一样需要 Big Picture 覆盖层来显示。

### SIAPI 模式输入样式（SIAPI Mode）

如果游戏原生实现了 SIAPI，则会提供游戏自定义的输入样式。大部分传统模式下可见的样式会被隐藏，但在启用“高级设置”后仍可显示并调整。Valve 也为 SIAPI 提供了对某些旧样式的重新实现（例如为 joystick-like 风格提供 Mouse Joystick 伴随样式），以补偿个别游戏未按最佳实践实现 SIAPI 的情况。
