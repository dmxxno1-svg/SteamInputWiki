# 02b11：径向菜单（Radial Menu）

此输入风格允许用户创建各种大小与透明度的径向菜单（圆形菜单），用于快速选择绑定动作。径向菜单的工作方式类似现代游戏中的武器轮（weapon wheel），可用于任何想要绑定的操作。

径向菜单可以包含任意可用的绑定与不同类型的 Activators（激活器）。常见用途包括武器选择、技能释放及热键。每个动作可以设置图标与名称，渲染时显示在菜单上。

适用设备：
* 触控板
* 摇杆
* 传统方向键（D-Pad）
* 传统面部按键簇（Face Button Pad）

在触控板与摇杆上，径向菜单按预期工作：玩家向某一方向输入以选择该方向对应的动作。

在面部按键或方向键上，工作方式不同，某些激活样式（如释放/点击）可能不按预期工作，通常仅用于给按键添加屏幕显示。

## 菜单按钮（最多 20 个）

菜单按钮即要放入径向菜单的动作。基于激活样式不同，选择方式也会有所变化，可使用不同 Activators 来改变分组绑定在激活时的行为。

对于基于按键的输入源，建议最多使用 8 个菜单按钮；尽管理论上可更多，但要选择非正方向项通常需组合两个按键，实际可用性受限。

每个按钮可设置名称与菜单图标（Menu Icon），图标会渲染在径向菜单上并可自定义颜色。

（提示：在游戏目录根目录下为该游戏创建名为 `touchmenuicons` 的文件夹并放入自定义图标，这些图标会只对该游戏生效并覆盖可用图标。）

### 嵌套触摸/径向菜单

可通过添加动作层或额外动作集将一个径向菜单嵌套在另一个中：在一个菜单中绑定“应用该层”的动作，在另一个菜单中绑定“移除该层”的动作，从而实现嵌套与循环菜单系统。

推荐视频：
Radial Menus: Nesting and Cycling - By Critical Input
Two Pad Menu System: An Adaptation of Nested Menus - By Critical Input

## 菜单激活样式（Menu Activation Styles）

决定如何从径向菜单中选择绑定：

* Button Click（按钮点击）——按下方向对应的点击以选择；对按键输入无额外意义因必须点击才能选择。
* Button Release（释放选择）——释放时选择绑定。触控板与摇杆可用，但按键输入在非正方向选择上精度不足。
* Touch Release / Modeshift End（触摸释放/模式切换结束）——触摸释放或模式切换结束时触发选择。
* Always（始终）——只要朝向某方向触发相应绑定即会立即执行。注意：使用该模式时建议仅将易于重复触发的动作绑定在菜单上。

## 径向菜单的视觉设置

* 屏幕水平/垂直位置（On-Screen Horizontal / Vertical Position）——调整菜单在屏幕上的显示位置；
* 菜单不透明度（Menu Opacity）——调整透明度；
* 菜单尺寸（Menu Size）——调整渲染大小；
* 在按钮上显示绑定标签（Display Binding Label on Button）——控制是否在图标上显示绑定的实际名称（名称源自输入项名，而非用户为绑定起的别名）。

## 额外按钮

* 中心/未选中按钮（Center/Unselected Button）——当未选择任何方向或选择中心时触发；可设图标与名称；
* 点击动作（Click Action）——当使用触控类输入选择菜单项时，触发点击动作（触控板或摇杆按下时触发；按键类在任意使用时触发）。

## 开发者实现

即便游戏原生支持 Steam Input，开发者仍可在发布配置中使用径向菜单。虽然不及游戏内定制的径向菜单漂亮，但实现更快，且可以随发行包一起包含自定义菜单图标。

# 02b11：环形菜单（Radial Menu）
